Money in software development is in:

  • Good requirements
  • Good implementations

The choice of the programming language is close to irrelevant to fulfill a set of requirements with a good implementation, i.e., algorithm. A particular language, most importantly, the runtime where the code will, eventually, run in and its included libraries, might help one get there faster or in a "nicer" way, one might have preferences regarding syntax, paradigma, etc., but all that has little impact on end value. There is, virtually, nothing only one language "does" or one language "doesn't".

Having that in mind, here goes an exploration of the basics, similarities and differences of some of the most used programming languages. Simple to complex examples implemented parallelly on each language to outline its implementation of the intended funcitonalilty. It might also serve as a quick cheatsheet.

Notes:

  • No libraries other than the ones included in the runtime or the operating system are used.
  • As a consequence of the previous point, the implementations are not necessarily the common practise.
  • The code here was tested on Windows (10), macOS (High Sierra) and Linux (Ubuntu 16.04 & 17.10). The documentation here is, for now, Windows driven. macOS and Linux installs will be included, in the future. Until then, refer to:
    • For Linux (Debian/Ubuntu) installs
    • For macOS installs
  • At least, an attempt is made to use the latest version of each language/runtime. The version used indicated in the list below.
  • The IDE used to code everything here (inlcuding this website) is Visual Studio Code for its code highlighting.
